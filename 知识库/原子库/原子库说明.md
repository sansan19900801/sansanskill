# 朋友圈案例原子库说明

`cases.jsonl` 每行是一条完整案例原子，由 `tools/build-atom-library.py` 从原始 Markdown 生成。

核心字段：

- `id`：稳定唯一编号；
- `original_type`：原始八分类；
- `route_type`：使用者看到的五分类；
- `title`：案例标题；
- `source_file`、`source_heading`：原文定位；
- `original_copy`：案例中的原成品文案；
- `sections`：全部结构化章节；
- `raw_markdown`：完整案例块，确保信息不丢失；
- `rights_status`：权利状态；
- `product_snapshot_status`：是否包含历史产品或价格口径；
- `case_sha256`：案例完整性校验值。

不要直接编辑 `cases.jsonl`。需要修改时编辑 `sources/朋友圈案例原文`，然后重新构建。

案例权利和发布状态在 `case-metadata.json` 中人工维护，例如：

```json
{
  "PYC-FDB-001": {
    "rights_status": "owned",
    "publishable": true
  }
}
```
