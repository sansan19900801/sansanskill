# 六类商业对标维度

根据用户当前目标选择一类主对标；只有用户明确要求完整商业拆解时才展开全部六类。

## 内容对标

重点判断内容怎样吸引正确的人、建立信任并进入业务承接，不只统计播放量。

| 维度 | 检查问题 |
| --- | --- |
| 目标受众 | 内容明确在和谁说话？ |
| 核心问题 | 持续解决哪类具体问题？ |
| 内容支柱 | 观点、方法、案例、产品、生活和背书怎样组合？ |
| 选题来源 | 来自客户问题、业务事件、公开研究还是热点？ |
| 表达形式 | 图文、口播、直播、长文或朋友圈承担什么任务？ |
| 信任证据 | 使用案例、过程、数据和第三方反馈怎样出现？ |
| 行动入口 | 用户看完后被引导去哪里、做什么？ |
| 业务连接 | 内容与产品、咨询、社群和成交怎样衔接？ |

不得复制具体标题、逐字稿、视觉资产和个人故事。应学习选题机制、结构和承接逻辑。

## 产品对标

重点判断产品为谁解决什么问题，承诺、价格和交付是否形成一致结构。

| 维度 | 检查问题 |
| --- | --- |
| 客户阶段 | 客户已经具备什么、目前卡在哪里？ |
| 核心结果 | 客户购买后具体获得什么变化？ |
| 产品边界 | 包含什么、不包含什么？ |
| 价格与门槛 | 价格、付款方式和准入条件是什么？ |
| 产品阶梯 | 免费、低价、主力和高阶产品如何递进？ |
| 证明材料 | 用什么证据降低购买风险？ |
| 交付结构 | 结果如何通过课程、服务、工具和反馈实现？ |
| 单位经济性 | 可见的收入方式与主要成本是什么；哪些仍未知？ |

不得仅因产品名称、页面包装或权益数量相似就判断产品机制相同。

## 获客对标

重点判断陌生人怎样被吸引、筛选并进入可承接的客户路径。

| 维度 | 检查问题 |
| --- | --- |
| 流量来源 | 自然内容、搜索、直播、投放、转介绍或合作？ |
| 触发钩子 | 用户因为什么问题或结果停下来？ |
| 前端载体 | 内容、工具、Skill、资料、测评还是活动？ |
| 筛选机制 | 怎样排除不匹配人群并识别高意向客户？ |
| 承接路径 | 公域如何进入主页、私信、微信、社群或咨询？ |
| 跟进节奏 | 进入后如何持续建立信任？ |
| 可见指标 | 线索、有效咨询和成交有什么公开或一手证据？ |
| 依赖条件 | 是否依赖投放预算、平台红利、资源置换或特殊渠道？ |

不能把浏览量、粉丝增长直接当成有效获客结果。

## 成交对标

重点判断客户怎样从感兴趣走到愿意付费，以及这套机制是否可以被复用。

| 维度 | 检查问题 |
| --- | --- |
| 成交入口 | 私聊、咨询、直播、社群发售还是销售页面？ |
| 客户筛选 | 怎样确认需求、预算、决策权和执行意愿？ |
| 需求诊断 | 怎样让客户看见问题、代价和目标差距？ |
| 方案呈现 | 产品如何与客户当前问题对应？ |
| 信任证明 | 案例、过程、体验、承诺边界如何呈现？ |
| 异议处理 | 价格、效果、时间和信任顾虑怎样回应？ |
| 促成方式 | 如何明确下一步而不依赖压迫式成交？ |
| 后续跟进 | 未成交和已成交客户怎样继续承接？ |

区分销售表达与产品真实价值；不能只复制一句话术。

## 交付对标

重点判断产品如何稳定帮助客户得到阶段结果，并把经验沉淀成可复用资产。

| 维度 | 检查问题 |
| --- | --- |
| 入门诊断 | 开始时如何了解客户现状并设定目标？ |
| 里程碑 | 服务期内有哪些可以验收的阶段结果？ |
| 标准流程 | 哪些环节有SOP、模板、Skill或工作流？ |
| 个性化边界 | 哪些必须人工判断，哪些可以标准化？ |
| 反馈机制 | 客户怎样提交结果、获得纠偏并继续执行？ |
| 质量控制 | 如何检查交付是否真正有效？ |
| 资产沉淀 | 问题、案例和方法怎样进入知识库？ |
| 复购转介绍 | 客户为什么继续购买或推荐？ |

不能只比较课时、资料数量和群服务，要比较客户结果形成机制。

## 商业模式对标

重点判断一个生意怎样持续创造、交付和获取价值，并识别主要约束。

| 维度 | 检查问题 |
| --- | --- |
| 客户 | 谁在什么场景下愿意付费？ |
| 价值 | 购买的是结果、效率、风险降低、身份还是便利？ |
| 收入 | 一次性、订阅、服务、授权、分成或组合？ |
| 成本 | 人工、获客、技术、供应链和机会成本主要在哪里？ |
| 获客成交 | 客户怎样进入并完成购买？ |
| 交付复购 | 结果怎样产生，为什么再次购买？ |
| 核心资产 | 品牌、内容、客户关系、数据、方法、渠道还是网络效应？ |
| 关键约束 | 创始人时间、现金流、平台、团队、资质或交付容量？ |
| 可持续性 | 增长是否依赖短期红利，风险集中在哪里？ |

商业模式对标可以调用其他五类作为证据，但最终必须收敛到一个最值得学习的机制。

## 跨类型选择规则

- 内容好但没有可见承接，只能作为内容对标，不能自动升级为商业模式对标。
- 产品卖得好但交付证据不足，可以作为产品或成交对标，不能断言交付优秀。
- 学员结果好但获客很弱，可以作为交付对标，不代表适合学习其增长方式。
- 同一对象在不同类型中可以得到不同结论。
- 如果一个差异不会影响目标结果，不把它列为必须模仿项。
