# 商业原子收录模式

## 入口

用户只需说“新增原子、收录原子、把这条做成原子”，或直接提交文字、截图、链接、文件、视频与逐字稿。

不要求用户记字段，不先发长表。先解析现有材料，只对缺失且必要的信息一次问一个。

## 收集顺序

1. 原始内容：文字、截图、文件、链接、视频或逐字稿；
2. 作者与权利：三三原创、客户授权、外部内容或尚未确认；
3. 来源平台：视频号、抖音、小红书、公众号、朋友圈、课程、聊天、业务复盘或其他；
4. 原始链接与发布时间：没有也可以，保留本地原文并标记无公开链接或日期未知；
5. 公开权限：允许进入公开Skill、仅内部使用或暂不确定；
6. 若为案例或实验：补充当时动作、数据、结果、日期和证据；
7. 若为外部模型：记录作者、原始标题、链接与引用边界。

材料里已经能识别的信息不重复问。截图涉及客户时提醒匿名化，但不要求用户重新转述截图里清楚可见的文字。

## 原子分类

- `principle`：三三商业原则或稳定判断；
- `case`：真实客户诊断或业务案例；
- `experiment`：有动作、条件和结果的经营实验；
- `external-model`：外部老师或公开理论，只作外部模型。

## 入库前预览

先展示：

```text
原子编号：
原子类型：
标题：
核心判断：
原始来源：
适用条件：
不适用条件：
证据等级：
权利状态：
公开状态：
关联Skill：
```

只有用户明确说“确认入库”后才写文件。

## 写入流程

仓库可写时：

1. 运行 `python3 tools/add-business-atom.py --type <类型> --title "<标题>"`；
2. 将确认内容写入生成的 `sources/商业原子原文/*.md` 模板；
3. 运行 `python3 tools/build-business-atoms.py`；
4. 运行商业原子与Skill系统校验；
5. 报告新增ID、来源文件和是否进入公开知识包。

仓库不可写时，输出完整待入库Markdown，不声称已经保存。

## 权利状态

- `owned`：三三原创；
- `authorized`：已获公开授权；
- `external-reference`：外部参考，只限允许范围；
- `unknown`：尚未核实。

只有 `publishable: true` 且权利状态为 `owned` 或 `authorized` 的原子可以进入公开Skill知识包。
