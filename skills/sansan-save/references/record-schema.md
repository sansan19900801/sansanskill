# 三三商业诊断存档结构

使用 YAML frontmatter 和固定 Markdown 正文。用户已经确认“保存这次诊断”时，视为授权保存本次诊断中的必要经营数据。

```yaml
---
project: 项目原名
timestamp: 2026-07-16T15:30:00+08:00
title: 当前最大瓶颈是公域获客
source_skill: sansan-business-diagnosis
status: in-progress
next_skill:
sensitivity: internal
schema_version: 1
---
```

时间使用本地时区 ISO 8601。`next_skill` 只有在已经明确且确实存在时填写。

```markdown
## 用户主诉

保留用户原始问题的简短摘要，不美化。

## 已确认经营事实

- 当前产品：
- 目标客户：
- 获客与成交：
- 交付与复购：
- 当前目标：

只写用户确认或材料证明的事实。

## 当前诊断结论

- 当前最大瓶颈：
- 为什么是这个瓶颈：
- 为什么暂时不是其他问题：

## 关键证据

- 证据：
- 来源：
- 证据强度：强 / 中 / 弱

## 已排除或否决的方向

- 方向：
- 原因：

## 待验证假设

- 假设：
- 验证方式：
- 验证期限：

## AI介入判断

- 适合AI介入：
- 暂不适合AI介入：
- 仍需人工判断：

## 当前7天动作

- 动作：
- 负责人：
- 截止时间：
- 验收标准：

## 推翻当前判断的条件

出现什么新证据时必须重新诊断。

## 需要回流的数据

下一次继续前需要带回哪些实际结果。

## 推荐下一步

只记录已确认的下一步及原因；没有就写“暂无”。

## 备注

只保留对未来诊断真正有用的信息。
```

四类内容不得混写：

- 已知事实：用户确认或有证据；
- 诊断判断：基于事实得出的当前结论；
- 待验证假设：尚未被市场结果证明；
- 行动建议：准备执行并回流结果的动作。
