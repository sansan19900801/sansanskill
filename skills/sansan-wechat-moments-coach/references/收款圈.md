# 收款圈

## 什么时候使用

素材核心是使用者真实收到付款、订金、报名或复购。收款圈解释客户为什么购买以及服务如何承接，不只晒金额。

## 必要输入

第一轮只让使用者提供：

- 他是谁；
- 他为什么要购买使用者的产品。
- 付款、报名或成交聊天截图。

可以直接照着说：

> 他是一位______，因为______，所以购买了我的______。这是相关付款或成交截图。

没有相关截图时先要求补图，不生成收款圈。金额、款项性质和产品名称优先从付款或聊天截图提取；无法确认且准备写入正文时，再补问一个最关键的问题。

## 模板选择

### A. 客户购买原因型

```text
客户想解决的具体问题
→ 客户身份和当前情况
→ 客户亲口表达的困难
→ 产品为什么匹配
→ 真实报名或付款
→ 当前只是开始，不冒充结果
```

### B. 收款后启动交付型

```text
真实付款时间
→ 次日或约定时间完成的首个交付动作
→ 双方对节奏的真实共识
→ 使用者的服务态度
→ 不把启动速度写成结果速度
```

### C. 老客户复购型

```text
老客户再次选择
→ 过去交付中建立的信任
→ 本次产品解决的新问题
→ 准确款项和权益
→ 对长期信任的感谢
```

### D. 内容转化型

```text
哪条内容发布后
→ 客户从哪里进入
→ 支付什么金额购买什么
→ 内容中的哪项价值促成行动
→ 产品真实交付
```

不能仅凭时间接近就断定成交归因。

### E. 诊断成交型

```text
客户身份与目标
→ 原有卡点
→ 本次完成的诊断
→ 客户获得的清晰结果
→ 为什么报名当前产品
→ 客户原话
```

## 金额闸门

- 订金、定金、审核金和全款不能混写；
- 报名数字必须说明是累计人数、第几位还是新增人数；
- 收款不证明产品效果；
- 历史产品、价格和权益不得套用；
- 付款前不能省略知情、适配和服务边界。

## 评论区

可以写：

> 如果你也卡在【具体问题】，可以把你当前的情况发给我，我先帮你判断是否适合。

不要写：

> 聪明人不犹豫／不买就是认知低／最后几个名额但没有真实限制。
