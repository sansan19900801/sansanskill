# 方法圈

## 什么时候使用

素材核心是一个真实可执行的工具、步骤、Skill、智能体或业务工作流。方法圈让读者看见“具体怎么做”，不是只讲工具名称。

## 必要输入

- 原来遇到什么业务问题；
- 旧流程怎样做、卡在哪里；
- 新流程使用什么工具；
- 输入、处理和输出分别是什么；
- 谁负责人工检查；
- 实际改善了什么；
- 隐私、版权和工具依赖。

## 模板选择

### A. 工作流改造型

```text
旧流程中的具体摩擦
→ 为什么原解决方案仍不够好
→ 新工具或步骤
→ 输入、处理、输出
→ 使用者如何分层使用结果
→ 真实改善
→ 人工检查和边界
```

### B. 自制Skill或智能体型

```text
一个有趣或高频问题
→ 为什么制作这个工具
→ 放入哪些方法和资料
→ 工具实际完成什么任务
→ 一次真实输出
→ 哪些只能参考
→ 回到业务验证
```

## 写作要求

- 说清业务问题，不只炫耀工具；
- 说清工具输入和输出，不用“一键完成所有事”；
- AI结果必须人工核对；
- 不把工具演示写成商业结果；
- 处理客户和社群资料前先匿名化；
- 避免依赖单一品牌，必要时给出等价流程；
- 趣味命理等内容只能用于娱乐和反思，不能验证高风险决策。

## 配图

优先使用：流程图、前后对比、真实产物、匿名界面。不要只放模型Logo或一张无法证明结果的聊天截图。
