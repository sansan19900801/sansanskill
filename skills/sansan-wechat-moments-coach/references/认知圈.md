## 什么时候使用

素材核心是使用者对创业、AI、学习、内容、商业或生活的真实判断。认知圈用于建立立场和筛选同频者，不靠制造对立假装深刻。

## 必要输入

- 想表达的核心观点；
- 观点来自亲历、客户观察、数据还是外部理论；
- 一个具体场景、案例或反例；
- 适用条件和不适用情况；
- 希望读者思考或采取的动作。

## 模板选择

### A. 表面解释与真正变量

```text
高关注问题
→ 大家常见的解释
→ 使用者不同的判断
→ 一个具体行为或案例
→ 适用条件
→ 克制结论
```

### B. 个人踩坑反思

```text
自己过去怎么做
→ 当时为什么觉得对
→ 真实结果为什么不理想
→ 重新发现的变量
→ 现在怎样做
```

适合：多AI低效、追工具、旧方法失效。

### C. 服务观察提炼

```text
真实样本来源
→ 反复出现的现象
→ 两三个可能原因
→ 不能外推的边界
→ 给读者的低风险提醒
```

出现财务、健康和法律话题时必须谨慎。

### D. 人物或师承故事

```text
人物条件与行为反差
→ 真实关系和故事来源
→ 可验证动作
→ 使用者得到的启发
→ 不把个案写成群体规律
```

### E. AI品牌立场

```text
工具学习或趋势焦虑
→ 当前真正的业务问题
→ AI可以介入的具体环节
→ AI不能替代的条件
→ 一个真实应用证据
→ 品牌主张
```

可使用：AI是工具，真实业务改善才是结果。

### F. 新产品形态判断

```text
旧产品的具体局限
→ Skill或工作流降低什么门槛
→ 仍需教学和人工判断的部分
→ 新交付组合
→ 用问题邀请读者自检
```

## 观点来源

明确区分：

- 本人亲历；
- 客户案例；
- 个人推测；
- 广泛流传观点；
- 外部老师或理论。

不要把“适者生存、AI放大器、用AI的人替代不用AI的人”等通行观点写成使用者原创。

## 风格

- 标题可以有冲突，但正文必须补条件；
- 少用“错了、死了、一定、所有人、99%”；
- 不以羞辱、阶层比较和恐惧制造认同；
- 结尾可以是判断、提问或轻行动，不强制卖产品。
