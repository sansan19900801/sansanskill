## 什么时候使用

素材核心是客户或学员在学习、使用、交付或业务中的真实变化。反馈圈用证据建立信任，不负责凭空制造成功案例。

## 必要输入

使用者最少提供：

- 客户使用产品前处于什么痛苦状态；
- 使用产品后达到了什么理想状态。
- 客户真实反馈截图。

没有反馈截图时先要求补图，不生成反馈圈。Skill从截图中提取前后变化，并判断对应产品或服务、客户原话，以及当前结果属于体验、操作、作品、咨询、成交还是回款。

截图中无法确认产品或结果时，只补问最关键的一项，不让使用者重复整理截图里已经存在的信息，也不把“感觉不错”扩写成业务结果。

## 新手描述模板

当使用者选择反馈圈但还没有提供素材时，引导他按以下链路描述：

```text
客户使用产品前处于什么痛苦状态
→ 使用产品后达到了什么理想状态
→ 客户真实反馈截图
```

给出可直接照填的一句话：

> 客户使用前______；用了我的______之后，现在已经______。这是客户的真实反馈截图。

如果发送截图，提醒使用者遮盖客户姓名、头像、微信号、手机号和其他敏感信息。“理想状态”必须是真实达到的变化，不能把客户期待或产品承诺写成已发生结果。

## 模板选择

### A. 效率结果型

```text
真实时间或任务对比
→ 客户要完成的具体工作
→ 过去的真实成本
→ 本次使用AI的过程
→ 客户对速度和质量的原话
→ 节省时间后用于什么更重要的事
```

适合：写文案、整理资料、制作方案等明确任务。

### B. 首次应用型

```text
过去知道工具但不会用
→ 本次学习或带练内容
→ 当堂完成的第一次作品
→ 客户真实感受
→ 当前只证明完成第一步
```

适合：一节课、一次带练、第一次写出朋友圈。

### C. 教学体验型

```text
客户过去害怕或听不懂
→ 本次教学方式的差异
→ 真实课堂动作
→ 客户体验反馈
→ 课程为什么更容易进入实操
```

适合：即学即练、非技术人听懂、团体培训。

### D. 交付价值型

```text
课程或服务节点
→ 客户提出的真实问题
→ 老师完成的具体支持
→ 客户评价
→ 使用者的交付理念
→ 明确服务边界
```

适合：答疑、作业反馈、结营后的额外帮助。

### E. 团队复制型

```text
创始人个人会但团队不会
→ 学会的具体方法
→ 如何同步给团队
→ 哪位成员完成了什么
→ 获得的真实反馈
→ 当前复制范围和下一步
```

适合：代理、销售团队和标准化内容流程。

### F. 业务结果型

```text
准确周期与结果口径
→ 学员原有业务基础
→ 实际调整的环节
→ AI和人工分别做了什么
→ 客户数量、产品与回款构成
→ 适用条件与不可复制部分
```

适合：咨询、成交、回款和高金额案例。必须读取真实性与产品边界。

## 默认写法

- 标题优先使用客户真实变化，不使用“逆天、封神、开挂”替代事实。
- 正文先写具体过程，再写老师观点。
- 客户原话短而准确；不是逐字记录就用转述。
- 评论区可以邀请同类客户咨询，但不能保证复制结果。
- 配图优先选择匿名反馈、作品对比或数据证据。

## 禁止扩大

- 报名不能写成学会；
- 学会不能写成持续应用；
- 完成作品不能写成获客；
- 一次咨询不能写成稳定成交；
- 个案收入不能写成普遍收益。
