## 1. 事实层级

写作前把信息分成四层：

| 层级 | 定义 | 可以怎样写 |
| --- | --- | --- |
| 已确认事实 | 使用者亲历、记录可见或客户明确表达 | 直接陈述 |
| 客户原话 | 有聊天、录音或使用者确认的准确转述 | 加引号；非逐字则改为转述 |
| 合理推测 | 根据上下文可能成立但未确认 | 不写成事实；必要时标记“可能” |
| 缺失信息 | 会影响价格、承诺、结果或隐私 | 先问使用者 |

不得自动补充：时间、地点、客户身份、收入、动作、第三方评价、情绪、名额、付款、产品效果。

## 2. 结果层级

严格区分：

```text
产生兴趣
→ 听懂或思路清晰
→ 完成第一次操作
→ 完成作品
→ 持续应用
→ 获得咨询
→ 促进成交
→ 获得回款
→ 形成稳定系统
```

低层结果不能直接写成高层结果。例如：

- “写出一条朋友圈”不等于“会持续获客”；
- “报名课程”不等于“卖出高价产品”；
- “方向清晰”不等于“项目成立”；
- “建群启动”不等于“交付见效”；
- “收到一次咨询”不等于“团队稳定复制”。

## 3. 数字与金额

出现数字时必须确认口径：

- 人数：报名、到场、峰值在线、结束在线还是全程在线；
- 时间：预计时间、实际操作时间、初稿时间还是最终交付时间；
- 付款：订金、定金、审核金、预付款、全款还是尾款；
- 收入：签约额、销售额、应收、实收、回款、毛利还是净利润；
- 效率：任务范围、质量标准、比较基准和测量周期；
- 结果：一次个案、初步验证还是稳定复现。

没有口径时不要用精确数字制造权威。

## 4. 客户原话和隐私

### 原话

- 逐字准确时才使用引号；
- 只保留大意时写“她的意思是”；
- 一位客户的反馈不能写成“大家都说”；
- 不替客户定义痛点、动机和内心状态。

### 隐私

发布前遮盖：

- 姓名、头像、微信号、手机号；
- 订单号、账户、支付渠道；
- 公司、机构和可组合识别的身份；
- 孩子、家人、健康和实时位置；
- 客户商业策略、产品数据和未公开结果。

## 5. 产品与活动

生成产品内容前确认：

- 当前有效产品名称；
- 价格和款项性质；
- 服务周期、形式与边界；
- 适合和不适合人群；
- 名额是否来自真实容量；
- 活动时间和二维码是否仍有效；
- 是否包含退款、抵扣和授权规则。

案例中的历史产品、老师姓名、价格和权益只用于理解结构，不作为当前事实。

## 6. 禁止表达

除非有严格证据且符合法律与平台要求，不生成：

- “保证出单、保证变现、用了就见效”；
- “三天学会、一节课掌握、照着做就能成交”；
- “最后几个名额、今晚涨价”但事实不存在；
- “聪明人不犹豫、不买就是认知低”；
- “比你有钱的人都更拼”；
- “所有人、大部分人、99%的人”但没有样本；
- 医疗、投资、法律和收益的确定结论；
- 命理或AI能够验证商业方向正确。

## 7. 安全改写

| 高风险表达 | 更准确的表达 |
| --- | --- |
| 一节课就会 | 当堂在指导下完成了第一次操作 |
| 用了就出单 | 发布后收到了一次真实咨询 |
| 三周变现 X 万元 | 三周内获得 X 万元实收回款，口径与构成如下；实际使用前核实金额与回款口径 |
| 全团队都会 | 第一位团队成员已完成应用 |
| AI替你赚钱 | AI辅助完成具体任务，业务结果仍取决于产品和执行 |
| 一键灌顶 | 降低第一次应用和重复执行的门槛 |
| 聪明人从不犹豫 | 适合当前需求的人可以先了解具体内容 |
| 课程结束我永远都在 | 按约定周期提供答疑，额外支持以实际安排为准 |

无法安全改写时，向使用者索取事实，不编造替代。
