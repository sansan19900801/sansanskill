# 发售圈

## 什么时候使用

素材核心是正式发布或招募一个产品、活动、课程或服务。发售圈必须说清客户问题、产品承接、交付、价格和行动入口。

## 必要输入

- 当前有效产品名称；
- 目标客户和不适合人群；
- 解决的阶段问题；
- 真实案例或需求证据；
- 周期、形式、交付和边界；
- 价格、优惠和名额依据；
- 付款、退款和授权规则；
- 有效时间和行动入口。

## 核心模板

```text
从客户已经理解的基础问题切入
→ 展示已有真实结果或需求证据
→ 指出下一阶段仍未解决的问题
→ 解释新产品具体解决什么
→ 说明适合谁、不适合谁
→ 写清周期、交付和完成标准
→ 准确说明价格、名额和付款规则
→ 给出有效且低压力的行动入口
```

## 发售逻辑

产品不是知识点堆叠。每次必须回答：

1. 客户为什么现在需要；
2. 当前产品承接哪一阶段；
3. 客户完成后能获得什么可检查交付物；
4. 哪些结果仍取决于客户执行和业务基础；
5. 为什么这个价格和容量真实成立。

## 禁止

- 虚构原价再制造折扣；
- 没有容量依据却写前10名；
- 把审核金、订金和赠品混写；
- “手把手”暗示无限代做；
- “新人照着做就能出单”；
- “跑通闭环”但没有完成标准；
- 使用过期二维码、日期和历史产品。

发售文案出现任何未确认事实时，在发布前核对中明确列出，不自动补齐。
