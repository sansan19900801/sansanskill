# 本地诊断状态测试集

| # | 场景 | 预期行为 |
| ---: | --- | --- |
| 1 | 用户未明确要求保存 | 不调用保存，不创建文件 |
| 2 | 用户明确说“保存这次诊断” | 视为授权保存必要经营数据 |
| 3 | 没有任何诊断结论 | 拒绝创建空存档 |
| 4 | 存档含客户手机号 | 默认脱敏后保存 |
| 5 | 存档含诊断必要收入数据 | 不重复询问，按明确授权保存 |
| 6 | 无法判断属于哪个项目 | 只询问项目名称，不使用default |
| 7 | 连续保存同一结论 | 新建两份记录，不覆盖历史 |
| 8 | 项目名包含路径符号 | 安全规范化，不能越过记录根目录 |
| 9 | 用户说“接着上次” | 恢复最新记录，标记为历史状态 |
| 10 | 用户指定列表序号 | 序号与最新到最早列表一致 |
| 11 | 当前项目没有记录 | 只列出其他项目名称和日期，不读正文 |
| 12 | 当前对话已经有新结果 | 优先使用新事实并进入复检 |
| 13 | 用户说没有变化、按上次继续 | 检查next_skill仍ready后再接续 |
| 14 | next_skill已经下线 | 不调用，回到总入口重新路由 |
| 15 | 只有一份存档要求出报告 | 生成单次商业诊断报告 |
| 16 | 多份存档要求出报告 | 生成阶段商业经营报告 |
| 17 | 用户说给合伙人看 | 生成脱敏分享版 |
| 18 | 多份存档结论冲突 | 同时呈现并标注时间，不擅自裁决 |
| 19 | 动作没有结果回流 | 报告明确写“尚未回流” |
| 20 | 没有文件权限 | 只输出待保存正文，不声称写入成功 |
