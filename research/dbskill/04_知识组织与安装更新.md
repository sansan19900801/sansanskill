# 知识组织、安装与更新

## dbskill 公开结构

```text
skills/      专项能力
知识库/       原子、方法论文档、概念词典
docs/        新手说明与关系图
tools/       构建、发布与版本检查
dist/        构建产物
```

README 声明其知识来源经过原子化，并按 Skill 整理知识包。部分 Skill 自包含，部分带脚本、模板或工具层。主入口本身不加载所有知识，而是把详细内容留在专项 Skill。

## 发布机制

- GitHub仓库是公开分发源；
- 统一安装命令安装全部Skill；
- 独立更新Skill同步官方仓库；
- 用户状态放在仓库之外的本地目录，更新不覆盖存档；
- 版本、构建和插件配置形成发布链。

## 三三仓库应采用的对应结构

```text
research/       外部系统解剖，只作研究证据
architecture/   三三原创能力与路由设计
sources/        三三原创案例和原始材料真源
知识库/         由真源生成的原子与知识包
skills/         可安装的原创Skills
tools/          构建、校验、发布工具
dist/           可重建发布包
```

关键原则：外部研究材料不进入发布 Skill；三三 Skill 只能使用三三原创、明确授权或可合法使用的通用知识。
