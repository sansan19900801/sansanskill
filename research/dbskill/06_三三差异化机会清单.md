# 三三差异化机会清单

## 核心差异

dbskill擅长把商业、内容、行动和Agent任务拆成工具；三三要进一步回答“这个AI动作最终是否进入获客、成交或交付，并产生可验证结果”。

## 值得建设的原创能力

| 优先级 | 能力 | 三三差异化 | 验证证据 |
| --- | --- | --- | --- |
| P0 | 商业诊断 | 先检查问题是否成立，再找一个最大瓶颈、承接能力和AI适配点 | 诊断后7天动作是否改变业务数据 |
| P0 | 朋友圈内容 | 使用真实截图与案例，服务信任和成交 | 发布、私信、咨询、成交反馈 |
| P1 | 私聊成交操盘 | 同时给客户判断、策略和可发送回复 | 回复后客户推进状态 |
| P1 | AI业务机会地图 | 从业务流程识别AI介入点，不追工具 | 节省时间、提高质量或增加结果 |
| P1 | 内容资产流水线 | 一个真实问题沉淀成私域、公域和知识资产 | 复用次数与线索结果 |
| P1 | 产品与定价诊断 | 连接客户阶段结果、交付成本与升级路径 | 真实报价、成交与拒绝数据 |
| P2 | 获客实验设计 | 只设计可测渠道动作，不泛谈流量 | 合格线索量与成本 |
| P2 | 交付与复购重构 | 把客户结果、SOP、反馈和复购连接起来 | 完成率、结果、复购转介绍 |
| P2 | 经营状态与复盘 | 让每次判断可被结果推翻或升级 | 连续回填与规则更新 |

## 暂不优先

- 固定名人聊天室；
- 与目标用户无关的多Agent迁移产品；
- 只为展示数量的标题、排版和小工具；
- 没有三三案例支撑的心理诊断。
