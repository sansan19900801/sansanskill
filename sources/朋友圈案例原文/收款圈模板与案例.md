# 收款圈模板与案例｜整理中

> 状态：素材收集中。三三明确“收款圈发完了”后再统一去重、归纳和封装进正式 Skill。

## 案例 01｜收款后立即启动交付

### 案例类型

- 圈型：收款圈
- 子类型：成交结果、客户行动力与交付启动
- 素材性质：收款事实与次日启动陪跑群的真实交付记录
- 适用场景：客户完成付款后，双方按照已经确认的安排迅速进入交付，希望展示客户行动力和服务承接效率
- 配图建议：匿名化收款记录、陪跑群建立截图或首个交付动作；不得暴露客户姓名、机构、金额和联系方式

### 使用者提供的案例事实

- 客户是一位医美机构创始人。
- 客户昨天完成付款。
- 今天已经建立陪跑群并启动交付。
- 客户希望尽快进入一个新赛道。
- 双方对快速行动的节奏具有共识。

### 证据状态

- 当前只收到成品文案，没有收到匿名化收款记录、陪跑群截图、产品名称或交付约定。
- “昨天收款、今天建群”和客户身份作为使用者提供的案例事实保存，正式公开前需确认真实可公开。

### 原成品文案

> 昨天收款，今天开干，这就是效率
>
> 昨天刚收的医美机构创始人的学员，今天陪跑群已经拉起来了。
>
> 效率这事儿，从来不是靠催出来的，而是双方都明白时间值钱
>
> 她急着在新赛道抢跑，我急着帮她落地，一拍即合，没有半点拖泥带水
>
> 说实话，我特别喜欢这种节奏——不观望、不犹豫，认准了就立刻行动
>
> 很多人输就输在“再等等”“再看看”，等机会过去了，连背影都抓不着

### 可复用结构

```text
用收款与首个交付动作的真实时间差作标题
→ 交代客户可公开的身份和成交事实
→ 写出付款后已经启动的具体交付动作
→ 解释快速启动来自双方怎样的共识
→ 表达自己欣赏的合作节奏
→ 收束到及时行动的价值，但不贬低谨慎决策者
```

### 标题公式

```text
【真实付款时间】完成合作，【真实启动时间】开始【首个交付动作】
```

或：

```text
从【成交】到【正式启动】，我们只用了【真实时间】
```

### 这条文案有效的原因

1. “昨天收款、今天建群”是具体可感知的效率证据。
2. 内容不仅展示成交，还展示了付款后的承接动作，降低潜在客户对交付拖延的担心。
3. 用双方共同重视时间解释效率，不把速度完全包装成单方面能力。
4. 客户身份与行动节奏能够吸引重视落地、愿意执行的同类客户。

### 风险与使用边界

- “医美机构创始人”可能使客户被识别，必须确认授权或进一步模糊身份。
- 收款截图、群聊截图和客户资料必须遮盖姓名、头像、机构、金额、订单号及联系方式。
- “昨天收款、今天开干”必须与真实付款和启动时间一致，不能为了制造效率感调整时间线。
- 快速建群只是交付启动，不等于已经产生业务结果；不能将“启动快”写成“必然见效快”。
- 客户需要谨慎评估产品、合同、预算和风险是合理行为，不能把所有“再看看”都定义为失败或没执行力。
- “她急着抢跑”涉及客户动机，必须来自客户明确表达，不能由AI自行揣测。
- 不得用“机会马上消失”等未经证实的说法制造成交焦虑。
- Skill生成时必须根据真实服务安排判断启动时间，不能默认承诺所有客户付款次日建群。

### AI 生成时需要提取的变量

- 客户允许公开到什么程度的身份信息；
- 真实付款时间；
- 产品或服务名称；
- 付款后完成的第一个真实交付动作；
- 交付正式启动时间；
- 双方对节奏和目标的真实共识；
- 客户亲口表达的行动原因；
- 使用者欣赏的合作特征；
- 该次快速启动是否为常规承诺还是个别安排；
- 可以公开使用的匿名收款和交付证据。

## 案例 02｜老学员基于历史信任提前支付订金

### 案例类型

- 圈型：收款圈
- 子类型：老学员复购、预热期成交与长期信任
- 素材性质：课程尚未正式宣传时，老学员主动支付订金锁定参与资格
- 适用场景：真实老客户基于过去交付体验再次购买或预订新产品，希望展示复购事实和信任积累
- 配图建议：匿名化订金记录、老学员复购对话或正式产品信息；不得泄露付款人和订单资料

### 使用者提供的案例事实

- 晚上7点收到一位伙伴支付的1000元。
- 该款项用于锁定“AI成交高手班”的参与名额。
- 在尚未正式宣传时，已有两位老学员提前付款。
- 使用者认为成交主要来自过去交付积累的信任。
- 文案描述的交付为30天、一对一指导以及AI成交系统落地。

### 事实状态与现行口径

- 当前只收到成品文案，没有收到付款记录、产品说明、总价、订金规则和两位老学员的授权。
- “AI成交高手班”不属于当前商业总纲记录的现行四阶段产品名称，本案例按历史或待确认产品素材保存，不能直接用于三三当前对外宣传。
- 1000元从文案语义看是锁定名额的款项，必须根据真实交易规则写成订金、定金、预付款或全款，不能笼统当作完整销售额。

### 原成品文案

> AI高手班第二单，来得真快
>
> 晚上7点，又一位伙伴转了1000块过来，直接锁定了AI成交高手班的名额
>
> 说实话，这次开课我还没怎么正式宣传，已经有两位老学员抢先打款了
>
> 没有花哨的营销话术，没有复杂的套路，靠的就是过去一点一滴积累下来的信任
>
> 她们知道，我教的不是花架子，是真正能用、能落地、能帮团队复制的东西
>
> 30天，一对一指导，把AI成交系统跑通，让新人也能快速上手出单
>
> 感谢每一位毫不犹豫的信任，这份沉甸甸的托付，我会用结果来回报

### 可复用结构

```text
用第几笔真实成交或复购事实作标题
→ 写清付款时间、款项性质和对应产品
→ 交代尚未正式宣传但已有老客户行动的背景
→ 把成交原因连接到过去的真实交付和信任积累
→ 说明老客户认可的具体产品价值
→ 准确说明本期产品的真实交付内容和边界
→ 表达感谢，并以认真交付而非结果保证收尾
```

### 标题公式

```text
还没正式【宣传/发售】，第【真实序号】位老学员已经【真实购买动作】
```

或：

```text
老学员再次选择【产品】，我更看重的是这份长期信任
```

### 这条文案有效的原因

1. 老学员在正式宣传前付款，能体现复购和既有信任，而不只是陌生客户的一次冲动成交。
2. 内容把收款连接到过去的长期交付，降低“单纯炫耀收入”的感觉。
3. 写出了产品希望解决的AI成交与团队复制问题，潜在客户能理解购买方向。
4. 结尾表达对信任的重视，有利于强化负责的交付形象。

### 风险与使用边界

- 必须准确区分订金、定金、预付款和全款，并确保退款、抵扣和锁位规则与真实交易一致。
- “第二单”“两位老学员”“晚上7点”“1000元”都必须来自真实记录，不能为了热销感调整数字和时间。
- “抢先打款”“毫不犹豫”涉及客户动机和心理状态，只有客户明确表达时才能使用。
- “没有正式宣传”需要符合真实发售状态；已经在私域预热、邀约或销售时不能包装成完全没宣传。
- “能落地、能帮团队复制”属于能力范围描述，需要真实案例或交付设计支撑。
- “把AI成交系统跑通”“让新人快速上手出单”容易被理解为确定结果承诺，必须改成明确的学习目标、陪跑动作和结果边界。
- “我会用结果来回报”可能形成对外结果保证，更稳妥的是承诺真实交付、认真反馈和按约定陪跑。
- 历史产品名称、价格、周期和权益不得自动套用到三三当前¥3,980产品或其他现行产品。

### AI 生成时需要提取的变量

- 当前有效的产品名称；
- 付款人是新客户还是老学员；
- 真实付款时间与成交序号；
- 款项是订金、定金、预付款还是全款；
- 产品总价及锁位、抵扣、退款规则；
- 发售前实际已经做过哪些宣传或邀约；
- 老学员再次选择的真实原因；
- 过去交付中积累信任的具体事实；
- 当前产品的真实周期、形式、交付内容和结果边界；
- 可以公开使用的匿名付款与复购证据。

## 案例 03｜产品消息发布后学员直接支付审核金

### 案例类型

- 圈型：收款圈
- 子类型：发布后快速付款、老学员信任与低咨询成交
- 素材性质：产品消息发出后，一位学员未进一步咨询便支付一笔用于报名审核的款项
- 适用场景：真实客户已经充分了解使用者及其服务，在看到新产品消息后主动采取付款行动，希望展示长期信任带来的低沟通成交
- 配图建议：匿名化付款记录和报名原话；必须遮盖姓名、头像、订单号及账户信息

### 使用者提供的案例事实

- 昨晚发布了“AI成交高手班”的相关消息。
- 一位学员随后支付1000元审核金。
- 学员表示：“大贝贝老师，我要报那个高手班。”
- 付款前没有进行额外咨询。
- 使用者将这次直接选择理解为长期信任的体现。

### 事实状态与现行口径

- 当前只收到成品文案，没有收到产品消息、付款记录、审核金规则或学员授权。
- “AI成交高手班”按历史或待确认产品素材保存，不能作为三三当前现行产品直接使用。
- “审核金”不是含义天然明确的标准产品口径，必须核对它是否可退、是否抵扣学费、审核未通过如何处理，以及付款前客户是否已知晓规则。

### 原成品文案

> 刚发布就收款，这就是信任的重量
>
> 昨晚刚把AI成交高手班的消息发出去，还没来得及正式宣传
>
> 手机就“叮”地响了……
>
> 打开一看，一位学员直接转了1000块审核金过来：“大贝贝老师，我要报那个高手班。”
>
> 没有咨询，直接打款
>
> 这种被笃定选择的感觉，比赚多少钱都让人踏实

### 可复用结构

```text
用发布与付款之间的真实时间反差作标题
→ 交代产品消息刚发出的背景
→ 用一个真实通知或查看动作制造现场感
→ 写清客户支付的金额、款项性质和报名原话
→ 说明成交前实际发生或没有发生哪些沟通
→ 把收款意义落到长期信任，而不是只展示金额
```

### 标题公式

```text
【产品消息】刚发出，就收到【真实客户行动】：这是长期信任的回音
```

或：

```text
没有反复沟通，老学员看到【产品】后直接【真实行动】
```

### 这条文案有效的原因

1. “刚发布就付款”的时间反差能直观呈现行动速度。
2. 付款原话和通知动作增加现场感，比单独晒收款数字更完整。
3. 内容把成交归因于长期信任，弱化单纯炫耀收入的感觉。
4. 篇幅短、信息集中，适合与真实付款截图配合发布。

### 风险与使用边界

- 已经发布产品消息本身就是宣传行为，不能同时声称“完全没有宣传”；更准确的是“尚未进行完整发售或集中宣传”。
- “刚发布”“直接付款”“没有咨询”必须与真实时间线和聊天记录一致。
- 引号里的报名话语必须是学员原话；不能为了增强笃定感重写成更强烈的直接引语。
- “审核金”必须有事先公开、清晰且合法合规的规则，不能用模糊款项名称规避订金、退款或合同义务。
- 没有咨询不代表可以省略产品说明、适配判断、价格确认、服务协议和风险边界。
- 收款速度只证明客户采取了付款行动，不证明产品效果、交付质量或最终结果。
- “比赚多少钱都踏实”属于使用者个人感受，可以保留，但不能用来遮盖真实商业交易属性。
- 历史老师姓名、产品名和收费方式不得自动迁移到三三当前业务。

### AI 生成时需要提取的变量

- 当前有效产品名称；
- 产品消息实际发布的时间和渠道；
- 付款发生的真实时间；
- 客户是新客户、老学员还是其他关系；
- 客户付款前已掌握哪些产品信息；
- 支付款项的准确名称、金额、用途、抵扣和退款规则；
- 客户真实报名原话；
- 付款前后实际发生的沟通；
- 使用者从这次行动中感受到的真实意义；
- 可以公开使用的匿名付款和聊天证据。

## 案例 04｜业务方向梳理后报名3980实战班

### 案例类型

- 圈型：收款圈
- 子类型：诊断成交、方向清晰与现行产品报名
- 素材性质：客户原本想进入AI教育赛道但缺少落地路径，接受现状梳理后报名3980AI落地实战班
- 适用场景：客户已经有真实业务或创业基础，通过一次诊断明确了适合自己的AI应用方向，并在了解产品后主动完成付款
- 配图建议：匿名化付款记录、客户关于方向清晰的原话或诊断交付物局部；不得公开客户商业机密

### 使用者提供的案例事实

- 客户是一位创业公司合伙人。
- 客户希望进入AI教育赛道，但此前不知道如何落地。
- 使用者当天为她梳理了现状并提供AI业务应用方向。
- 客户随后支付3980元，报名AI落地实战班。
- 客户反馈：“终于知道方向在哪了！”

### 证据状态与产品边界

- 当前只收到成品文案，没有收到付款记录、诊断内容、产品说明或客户授权。
- 3980元与三三当前AI落地实战班现行价格一致，但仍需核对本笔付款和产品名称。
- “用AI重构商业系统的一套打法”可能超出3980产品边界；如果涉及商业战略、产品体系和完整系统重构，应属于19800层级。

### 原成品文案

> AI＋教育赛道怎么落地？她今天直接转了3980
>
> 她是个创业公司合伙人，一直想做AI教育赛道，但完全没方向，不知道怎么落地
>
> 今天我帮她梳理了现状，给出一套用AI重构她商业系统的打法，
>
> 她听完直接报了我的3980AI落地实战班！
>
> 她说：终于知道方向在哪了！

### 可复用结构

```text
用客户问题和真实付款结果作标题
→ 交代客户身份、已有业务与目标
→ 写客户原来卡住的具体落地问题
→ 说明本次完成了哪些诊断和梳理
→ 写清客户为什么认为现行产品能承接下一步
→ 准确呈现付款金额和产品名称
→ 用客户关于清晰度的真实原话收尾
```

### 标题公式

```text
【客户类型】一直卡在【具体问题】，梳理清楚后她报名了【真实产品】
```

或：

```text
从“不知道怎么落地”到明确下一步，她今天完成了【真实购买动作】
```

### 这条文案有效的原因

1. 客户问题、诊断动作和付款结果形成完整成交路径。
2. 创业公司合伙人的身份能吸引有真实业务基础的目标客户。
3. “终于知道方向在哪了”具体呈现诊断带来的清晰度价值。
4. 使用现行3980产品和价格，具备直接承接当前业务的潜力。

### 风险与使用边界

- “创业公司合伙人”可能使客户被识别，应获得授权或进一步模糊身份。
- “完全没方向”容易贬低客户已有探索，优先使用客户自己的准确表达。
- “直接转了3980”必须有真实付款记录，并确认是全款而非订金或其他款项。
- 客户付款前仍需了解产品内容、服务周期、适配条件和退款规则，不能用“听完就付款”省略必要的知情过程。
- “重构商业系统的一套打法”范围过大，3980产品应准确写成业务场景诊断、AI应用路径、Skill定制或实操支持。
- 客户说方向清楚了，只证明认知清晰，不证明AI教育项目已经成立或能够盈利。
- 不能因为客户报名，就把诊断建议写成已经过市场验证的商业方向。
- 如果客户尚无清晰产品、客户或真实业务基础，需要先判断是否符合3980招收条件。
