# 认知圈模板与案例｜整理中

> 状态：素材收集中。三三明确“认知圈发完了”后再统一去重、归纳和封装进正式 Skill。

## 案例 01｜用关键时刻的行为判断创业者韧性

### 案例类型

- 圈型：认知圈
- 素材性质：专业观点与价值观表达
- 适用场景：针对一个经常被讨论的问题，提出鲜明判断标准，并用行为对比解释自己的创业观或做事原则
- 配图建议：可不配图；也可搭配本人工作、解决问题或长期行动的真实场景图

### 核心观点

- 一个人是否具备创业所需的韧性，关键要看他遇到困难时是退出、抱怨，还是继续寻找解决办法。
- 开始时的热情不能证明长期行动能力，真正的差异会在遇到阻力后出现。

### 证据状态

- 当前收到的是观点型成品文案，没有附带数据、具体人物案例或个人经历。
- 文中的“大部分人”“极少数人”属于修辞性概括，不应被当作经过统计验证的事实。

### 原成品文案

> 创业能不能成，看这一点就够了
>
> 经常有人问我，怎么判断一个人适不适合创业？
>
> 其实不用看太多，就一个点：遇到困难时的反应
>
> 刚开始的时候，谁不是激情满满、斗志昂扬？
>
> 那叫新鲜劲儿，不算本事
>
> 真正的分水岭在遇到第一个坎儿的时候
>
> 大部分人开始抱怨、犹豫、慢慢躺平，最后悄无声息地退场
>
> 但总有极少数人，碰了壁也不慌，脑子里想的是“这事还能怎么解决？”
>
> 熬夜查资料、反复试错、厚着脸皮请教人，硬是把问题啃下来
>
> 这种人，才真正适合创业
>
> 创业不是比谁开头喊得响，是比谁在黑暗里还能摸着往前走
>
> 扛得住事，才接得住结果

### 可复用结构

```text
用一个高关注问题作标题
→ 提出读者经常讨论或困惑的问题
→ 给出一个鲜明的核心判断标准
→ 排除一个容易被误认为关键的表面信号
→ 指出真正的检验时刻
→ 对比两种人在关键时刻的不同反应
→ 用具体动作描写支撑正面类型
→ 回到个人立场并用一句金句收束
```

### 标题公式

```text
【重要结果】能不能实现，关键看【一个可观察行为】
```

或：

```text
怎么判断一个人能不能【目标】？看他【关键场景下的行为】
```

### 这条文案有效的原因

1. 从“适不适合创业”这一高关注问题切入，读者容易代入和自我判断。
2. 用“初始热情”和“遇到困难后的反应”制造清晰反差。
3. 不只讲抽象的韧性，而是用查资料、试错和请教等动作把观点落地。
4. 结尾连续使用两句短句，形成容易记忆和传播的价值主张。

### 风险与使用边界

- “看这一点就够了”是强钩子，不是严谨的创业成功判断模型；正式表达时可改为“我最看重这一点”。
- 不把创业成功简化成单一人格品质，还需要考虑产品、市场、资源、能力和时机。
- “大部分人”“极少数人”没有数据支持时，优先改为“有些人”“也有人”。
- “这种人才真正适合创业”容易把观点写成绝对裁决，可改为“这种反应，至少说明他具备创业很需要的一种能力”。
- 不把熬夜包装成创业者必须具备的品质；核心应该是主动解决问题，而不是牺牲健康。
- 引号里的心理活动如果不是具体人物原话，应当明确作为典型思路表达，不伪装成真实引用。
- AI只能协助表达使用者真正认可的观点，不能为了制造冲突自动生成与使用者价值观不一致的立场。

### AI 生成时需要提取的变量

- 使用者想讨论的高关注问题；
- 使用者真正认可的核心判断；
- 一个常被误认为关键、但使用者认为不够重要的表面信号；
- 能检验核心判断的真实场景；
- 两类人在该场景中的不同反应；
- 支撑观点的具体动作、经历或观察；
- 使用者愿意公开承担的明确立场；
- 最终希望读者记住的一句话。

## 案例 02｜AI竞争认知带动活动引流

### 案例类型

- 圈型：认知圈（活动引流型）
- 素材性质：趋势观点、竞争提醒与直播活动招募
- 适用场景：先表达与业务相关的认知观点，再承接到一个真实存在、时间明确、内容清楚的公开课、直播或体验活动
- 暂定分类：先作为认知圈的转化型变体收录；若后续出现多条同类案例，再评估是否独立为“活动圈/引流圈”
- 配图建议：真实活动海报、直播主题图或包含有效二维码的招募图

### 核心观点

- AI本身不会主动抢走某个人的工作，但更早把AI用于实际业务的同行可能获得效率和竞争优势。
- 使用AI的价值不只在于做得更快，也在于把时间释放到策略、客户和产品等更高价值环节。

### 使用者提供的活动信息

- 活动形式：AI实操快闪群及直播教学。
- 活动时间：明晚8点。
- 活动内容：讲解几个可以立即使用的AI提效技巧。
- 行动方式：扫描二维码进群。

### 证据状态

- 当前只收到成品文案，没有收到活动海报、二维码、直播大纲或效果验证数据。
- “十分钟干完一天的活”“干活快一倍”等效果表达当前没有证据支持，不能作为通用活动承诺。

### 原成品文案

> 替代你的不是AI，是用AI的同行
>
> 很多人担心AI会抢饭碗，说实话，这担心有点偏了
>
> 真相是，AI不会主动替代你，但那个偷偷用AI的同行会替代你
>
> 你还在手动写文案、憋方案、熬通宵的时候
>
> 人家早就用AI十分钟干完你一天的活
>
> 剩下的时间用来思考策略、链接客户、迭代产品
>
> 时间一长，差距就出来了
>
> 这个时代，不是你要不要选AI的问题，而是你要不要被同行甩开的问题
>
> 不想掉队的话，立刻扫码进我的AI实操快闪群
>
> 明晚8点，我直播教你几个立刻就能用的AI提效技巧
>
> 让你干活快一倍，轻松甩开同行
>
> 别等人家跑远了，你还在原地纠结，扫码占位，明天见！

### 可复用结构

```text
用一句反常识判断提出主题
→ 纠正受众常见但不准确的担忧
→ 描述使用与不使用工具的工作方式差异
→ 把效率优势连接到更高价值的业务动作
→ 提醒长期积累可能形成差距
→ 自然承接到一场真实存在的低门槛活动
→ 说清活动时间、内容和参加方式
→ 给出明确行动指令
```

### 标题公式

```text
真正让你失去【结果/优势】的不是【表面对象】，而是【更具体的竞争变化】
```

### 这条文案有效的原因

1. 标题用认知反转快速建立冲突，能吸引对AI既焦虑又犹豫的人。
2. 中段把AI提效连接到策略、客户和产品，没有只停留在工具炫技。
3. 活动时间、教学内容和扫码动作比较明确，读者知道下一步做什么。
4. 认知内容和活动主题一致，转化动作不是突然插入的无关广告。

### 风险与使用边界

- “替代你的不是AI，是用AI的同行”属于广泛传播的观点表达，不应包装成三三独创理论。
- “十分钟干完你一天的活”“干活快一倍”属于可量化效果承诺，只有在任务范围、对比条件和真实结果明确时才能使用。
- “轻松甩开同行”既缺乏证据，也容易把AI使用写成简单竞争捷径，正式模板中不建议保留。
- “被同行甩开”“不想掉队”“别等人家跑远了”连续叠加会形成恐惧驱动，与三三“不靠恐惧营销”的现行原则冲突。
- 不得虚构活动时间、直播内容、二维码、群名额或报名截止时间。
- 活动已经过期时，不得继续生成“明晚8点”等失效信息，必须向使用者确认最新活动事实。
- 更稳妥的转化逻辑应当是让读者看见真实效率差距和可尝试的第一步，而不是承诺学完就能超过同行。

### AI 生成时需要提取的变量

- 使用者真正认可且愿意公开承担的AI观点；
- 目标客户当前仍在手工完成的具体任务；
- AI可以真实缩短或改善的业务步骤；
- 节省时间后可以投入的更高价值工作；
- 活动名称、形式、时间和入口；
- 活动会真实讲授或带做的内容；
- 已有的效率结果证据及适用条件；
- 不依靠恐惧和夸张承诺的行动理由。

## 案例 03｜从一对一咨询样本提炼守钱认知

### 案例类型

- 圈型：认知圈（咨询观察型）
- 素材性质：真实服务观察、反差案例与财务认知表达
- 适用场景：使用者在一批真实咨询、服务或访谈中反复看到同一种问题，希望在保护客户隐私的前提下提炼可供读者警醒的规律
- 配图建议：本人咨询工作场景、匿名化咨询记录数量或相关手写复盘；不展示客户身份和具体资产信息

### 使用者提供的观察

- 最近为20多位创业者提供了一对一咨询。
- 部分咨询者曾在高峰期一年收入达到大几百万元。
- 部分咨询者当前连5000元或1万元的可用资金都拿不出来。
- 使用者将原因概括为：高收入阶段没有储蓄，以及有钱后盲目投资。

### 证据状态

- 当前只收到成品文案，没有收到匿名化咨询记录、样本统计方法或客户授权说明。
- “20多位”“大几百万”“拿不出5000或1万元”作为使用者提供的服务观察保存，不视为可以外推到所有创业者的统计结论。

### 原成品文案

> 赚钱靠运气，守住钱靠本事
>
> 最近给20多位创业者做一对一咨询，打完电话心情挺沉重的
>
> 他们当中有人一年高峰期赚过大几百万，可现在连5000、一万块都拿不出来
>
> 为什么会这样？我总结了两个扎心的真相。
>
> 第一，赚钱的时候没学会存钱
>
> 总觉得好日子长着呢，今天花明天的，结果风口一过，手里干干净净
>
> 记住，赚钱的机会就那么几年，不可能年年都是高峰期，有钱的时候一定要留后手。
>
> 第二，有钱了就盲目投资
>
> 看这个项目好投一点，那个风口跟一把，结果一次失误，把好几年赚的钱全亏完了
>
> 想再赚回来，难如登天
>
> 所以，赚钱靠运气，守住钱靠本事，别等到手头紧了才后悔当初没留后路

### 可复用结构

```text
用一句有张力的对比观点作标题
→ 交代观点来自哪一批真实服务或观察
→ 呈现一个前后落差明显的匿名化现象
→ 提出“为什么会这样”的核心问题
→ 提炼两个或三个反复出现的原因
→ 每个原因配一个具体行为链条
→ 给出克制、可执行的提醒
→ 回扣标题完成收束
```

### 标题公式

```text
得到【结果】可能靠【外部机会】，守住【结果】更需要【长期能力】
```

或：

```text
真正难的不是【获得】，而是【长期保有】
```

### 这条文案有效的原因

1. “赚过大几百万”和“拿不出5000元”的强烈反差能快速建立问题意识。
2. 观点来自一对一咨询场景，比纯粹说教更具有专业观察感。
3. 把复杂的财务困境拆成储蓄和投资两个容易理解的行为问题。
4. 标题与结尾首尾呼应，观点清楚且容易记忆。

### 风险与使用边界

- 客户收入、现金状况和投资亏损属于高度敏感信息，必须充分匿名化，且不能组合出可识别的个人画像。
- “20多位”只有在确实完成对应数量的咨询后才能使用；不能为了增强权威感虚构样本。
- 个别咨询案例不能证明所有创业者的财务困境都由“不存钱、乱投资”造成，还可能涉及债务、家庭支出、经营现金流和行业周期。
- “赚钱靠运气”过度弱化能力、选择和长期积累，更准确的表达可以是“赚到阶段性高收入可能有机会因素，守住成果需要长期能力”。
- “赚钱的机会就那么几年”没有普遍证据，不应作为确定规律；可以改为“高峰期不会自动永远持续”。
- “一次失误把几年赚的钱全亏完”“难如登天”只能用于真实个案，不得由AI自行补充。
- 涉及储蓄、投资和资金安排时，只能表达一般性风险提醒，不代替针对个人情况的专业财务建议。
- 不利用客户困境制造羞耻或优越感，重点应放在风险意识和可执行的经营纪律。

### AI 生成时需要提取的变量

- 使用者真实服务或观察的样本来源；
- 可以公开且无法识别个人的样本数量；
- 反复出现的真实现象；
- 前后落差及其准确口径；
- 使用者观察到的可能原因；
- 哪些是已知事实，哪些只是解释或假设；
- 每个原因对应的具体行为链条；
- 使用者希望读者采取的低风险行动；
- 涉及财务、健康或法律等高风险领域时需要加入的边界说明。

## 案例 04｜高收入人物仍保持强执行力的反差观察

### 案例类型

- 圈型：认知圈（人物故事型）
- 素材性质：第三方人物故事、财富反差与行动价值观表达
- 适用场景：使用者掌握一个真实、得到授权且可以匿名公开的人物案例，希望通过该人物的具体行为表达自己对执行力、成事习惯或长期主义的理解
- 配图建议：经本人授权的工作场景、匿名化行动记录；不直接展示财富、收入和身份信息

### 使用者提供的故事信息

- 故事来自一位私教学员对其团队成员的描述。
- 该团队成员经济条件较好，本可以选择不工作。
- 她被描述为团队中执行力很强的人。
- 文案称她曾多次实现单日10万元纯收入。
- 她在高强度行动中很少公开喊累。

### 证据状态

- 当前只收到成品文案，没有收到当事人授权、收入记录、团队数据或一手访谈。
- 所有信息均属于“学员转述给使用者、再由使用者写成文案”的第三方材料，证据链较弱。
- “无数次日入10万纯收入”属于高度敏感且影响信任的财务结果，正式作为案例使用前必须核实准确口径并确认允许公开。

### 原成品文案

> 比你有钱的人，比你还拼
>
> 一位私教学员对我说，她团队里的许姐姐，其实是个富婆，完全可以不工作、享清福
>
> 可你猜怎么着？
>
> 她偏偏是团队里执行力最强的那个人，无数次做到日入10万纯收入，喊累的从来不是她
>
> 这事儿挺有意思的，现在大环境不好，一堆人喊着赚不到钱、直接躺平
>
> 而那些本身不缺钱的人，反而比谁都卖力地在干，你说奇怪不奇怪？
>
> 其实一点都不怪，真正优秀的人，早就把努力当成了习惯，跟缺不缺钱没关系
>
> 他们对成事有执念，对自我有要求，闲下来反而浑身不自在

### 可复用结构

```text
用人物条件和人物行动的反差作标题
→ 交代故事来源及人物关系
→ 写出人物原本可以选择不行动的条件
→ 用一个真实、可验证的行为说明其执行力
→ 对比常见的另一种反应
→ 提出“为什么会这样”的问题
→ 给出使用者真正认可的行为解释
→ 收束到执行力或长期习惯的认知观点
```

### 标题公式

```text
明明已经【拥有某种条件】，她为什么还在【持续行动】？
```

或：

```text
真正拉开差距的，可能不是【外部条件】，而是【长期行为】
```

### 这条文案有效的原因

1. “有钱却仍然拼”形成违背部分人直觉的反差，容易引发阅读兴趣。
2. 使用具体人物故事承载观点，比直接说“优秀的人都努力”更有画面。
3. 从金钱动机转向成事习惯，形成更深一层的价值解释。
4. 结尾把故事收束到行动和自我要求，观点比较明确。

### 风险与使用边界

- “许姐姐”“富婆”“日入10万纯收入”可能使人物被识别，涉及隐私和敏感财务信息，未经本人授权不能公开。
- 该故事来自学员转述，不是一手材料；AI必须标注信息来源，不能写成使用者亲眼见证。
- “无数次”是模糊且夸张的频率词，应替换为有证据的真实次数或删除。
- “纯收入”需要明确计算口径，不能把流水、销售额、团队业绩或毛利润写成个人纯收入。
- “喊累的从来不是她”“闲下来浑身不自在”涉及绝对化行为和内心推断，只有当事人明确表达时才能使用。
- 标题“比你有钱的人，比你还拼”容易通过比较制造羞耻，正式模板应优先把注意力放在可学习的行动，而不是贬低读者。
- “大环境不好”“一堆人躺平”属于未经界定的群体概括，不能为了衬托人物而贬低其他人。
- 财富与努力之间没有简单因果关系，个别人物故事不能证明“有钱人都更努力”或“不成功都是因为不够拼”。

### AI 生成时需要提取的变量

- 故事是一手经历、当事人原话还是第三方转述；
- 当事人是否授权公开，以及需要隐去哪些身份信息；
- 人物拥有的真实条件；
- 与该条件形成反差的真实行动；
- 能证明执行力的具体行为，而非只使用收入数字；
- 收入、业绩和利润等数据的准确口径；
- 使用者从故事中得到的真实启发；
- 可以被读者学习的行动，而不是用于羞辱比较的标签；
- 哪些内容是事实，哪些只是使用者的解释。

## 案例 05｜旧方法失效后的持续适应

### 案例类型

- 圈型：认知圈
- 子类型：趋势变化、方法迭代与生存认知
- 素材性质：从过去有效的方法效果下降出发，表达创业者需要持续观察用户和调整打法的观点
- 适用场景：使用者基于真实经营、内容或客户反馈，发现某种过去有效的方法在当前阶段明显衰减，希望提醒读者重新观察环境
- 配图建议：本人复盘、工作调整或新旧数据对比；没有真实证据时可不配图

### 核心观点

- 方法的有效性依赖环境、用户习惯和需求，不能假设一套打法永久有效。
- 调整方法不一定是盲目折腾，也可能是根据真实反馈进行必要适应。
- 长期经营需要保持敏锐，同时避免固守经验和无依据地频繁换方向。

### 证据状态

- 当前只收到观点型成品文案，没有收到具体赛道、旧方法、新方法或效果变化数据。
- “以前管用、现在大打折扣”作为使用者的经营感受保存，尚未说明观察范围和判断依据。

### 原成品文案

> 没有一招鲜，只有一直变
>
> 最近我有一个很深的感受：不管做哪个赛道
>
> 靠同一套打法吃到底的日子，早就过去了
>
> 以前超级管用的招，现在用效果却是大打折扣
>
> 不是我们不够努力，而是时代在变，用户的习惯和需求也在变
>
> 所以别守着旧方法不放，也别觉得“我过去就是这样做的”
>
> 真正能活下来的人，不是最厉害的那个，而是最愿意跟着变化走的那个人
>
> 保持敏锐，拥抱变化，随时调整自己的节奏
>
> 这不是折腾，是生存

### 可复用结构

```text
用“旧确定性失效”的判断作标题
→ 说明观点来自最近的真实感受或观察
→ 写出过去有效、现在衰减的具体现象
→ 分析变化可能来自哪些环境或用户因素
→ 指出固守旧经验的风险
→ 提出更值得保留的长期能力
→ 区分有依据的调整与无目的折腾
→ 用一句简短立场收束
```

### 标题公式

```text
没有永远有效的【方法】，真正需要保留的是【适应能力】
```

或：

```text
以前有效的【打法】正在失效，我开始重新看这件事
```

### 这条文案有效的原因

1. “一招鲜”和“一直变”形成简短对比，观点容易被记住。
2. 把方法失效与用户习惯、需求变化连接起来，不只归咎于个人努力。
3. 能击中依赖过去成功经验、但当前效果下降的创业者。
4. 结尾把调整解释为适应而非盲目折腾，强化行动立场。

### 风险与使用边界

- “不管做哪个赛道”“早就过去了”是跨行业绝对判断，没有足够证据时应改成个人观察范围。
- “以前超级管用、现在大打折扣”最好补充具体方法、时间区间和真实数据，否则观点容易空泛。
- “真正能活下来的人，不是最厉害而是最愿意变化的人”接近广泛流传的适者生存表达，不能包装成使用者原创理论。
- 愿意变化不等于频繁换赛道、追热点或推翻所有积累；调整应由客户反馈、转化数据和环境变化支持。
- 旧方法效果下降也可能来自执行质量、产品变化、渠道问题或样本波动，不能未经验证就归因于“时代变了”。
- “随时调整”容易导致短期主义，更稳妥的是设定观察周期、指标和停止条件后再迭代。
- AI生成时要给观点补充真实经营观察，不能只堆叠“拥抱变化、保持敏锐”等抽象口号。

### AI 生成时需要提取的变量

- 使用者观察的是哪个行业、渠道或具体业务场景；
- 过去有效的具体方法；
- 该方法过去和现在的真实表现；
- 对比数据的时间范围和口径；
- 用户习惯或需求发生了什么可观察变化；
- 还有哪些可能导致效果下降的解释；
- 使用者实际做了或准备做什么调整；
- 如何判断调整有效，何时停止继续变动；
- 使用者真正认可的长期能力；
- 哪些观点来自个人经验，哪些来自外部通行观点。

- 指导者与使用者的真实关系；
- 学习或合作持续了多久；
- 本次真实交流的时间和主题；
- 使用者在交流前后的真实感受；

- 使用者批评的是哪一种具体旧产品形态；

```text
→ 指出学完功能后仍未解决的业务问题
→ 提出使用者不同的教学关注点
→ 列举AI进入业务的具体场景
→ 用真实案例或数据说明应用价值
→ 说明工具基础与业务判断缺一不可
→ 用一句符合品牌定位的观点收束
```

### 标题公式

```text
学会【AI功能】以后，下一步怎么进入【具体业务结果】？
```

或：

```text
AI工具会用还不够，真正的差距发生在【业务环节】
```

### 这条文案有效的原因

1. “功能能不能换成业务结果”直接击中学了很多工具却没有收益的人。
2. 私聊、朋友圈、销售信和智能体都是具体场景，定位不是抽象的“AI赋能”。
3. 观点与三三“帮有产品的人把AI装进业务”的现行定位一致。
4. 结尾“AI不是拿来学的，是拿来用的”简洁，容易形成品牌记忆。

### 风险与使用边界

- “能当饭吃吗”“没人讲”“我这儿不整那些”容易通过贬低同行建立优越感，建议承认功能教学是业务应用的基础之一。
- “功能问AI两天就会”没有区分学员基础、工具复杂度和任务范围，不能作为统一学习周期承诺。
- “朋友圈怎么写能有咨询有出单”不能暗示文案本身保证咨询和成交，结果还取决于流量、信任、产品和承接。
- “销售信没有营销味”是风格目标，不等于销售信一定有效。
- “智能体省一半时间”需要明确任务、基准、质量标准和真实数据。
- “能用出钱来才算没白学”过度缩窄学习价值，也可能让尚处在能力建设期的学员产生不必要焦虑。
- AI应用不能绕开产品、定位、客户需求、真实交付和人工判断。
- 3980产品可以承诺场景诊断、工具应用、Skill定制和执行支持，不能无条件承诺客户必然多成交或变现。

### 与三三当前业务的适配判断

- 高度适配：AI不能停在工具演示，必须进入真实业务环节。
- 高度适配：围绕私聊成交、朋友圈、销售信和工作流形成具体应用。
- 需要调整：从“AI功能没价值”改为“功能是基础，业务应用才完成价值闭环”。
- 需要调整：从“能用出钱”扩展为可验证的阶段结果，例如提效、完成作品、获得咨询、促进成交或改善交付。
- 不可承诺：学员两天学会、文案必然出单、智能体统一省一半时间。

### AI 生成时需要提取的变量

- 目标客户已经学过哪些AI功能；
- 客户学完后仍卡在哪个业务环节；
- 本次内容要讨论获客、成交、内容、交付还是效率；
- AI在该环节完成什么具体任务；
- AI不能替代哪些商业条件和人工判断；
- 已有的提效、作品、咨询或成交证据；
- 数据的任务范围、时间周期和比较口径；
- 当前产品真实提供哪些诊断、教学、Skill或工作流服务；
- 当前阶段可以承诺什么，不能承诺什么；
- 如何在不贬低工具教学的前提下表达三三差异。

## 案例 09｜同时使用多个AI带来的无效效率

### 案例类型

- 圈型：认知圈
- 子类型：AI效率陷阱、内容判断与人机边界
- 素材性质：从自己同时开启多个AI生产内容的经历出发，反思产出数量增长并不等于可用内容增长
- 适用场景：使用者有真实的多工具、多智能体或批量生产经历，希望提醒读者把质量判断和业务目标放在工具数量之前
- 配图建议：真实工作台、被淘汰的稿件数量对比或人机协作流程图；注意隐藏账号和客户资料

### 使用者提供的个人经历

- 使用者过去曾同时开启多个AI，分别处理文案、脚本和朋友圈。
- 当时认为AI数量越多，产出速度越快。
- 实际得到大量稿件，但真正能够使用的内容很少。
- 使用者将问题归因于内容缺少人的感受和最终判断。
- 使用者认为，无效产出的加速并不等于真正提效。

### 证据状态

- 当前只收到观点型成品文案，没有收到实际工具数量、产稿数量、采用率或修改数据。
- “十几个AI”“能用的没几条”作为使用者提供的个人经验保存，正式发布时可补充更准确的真实数据。

### 原成品文案

> 同时开十几个AI的人，不一定真的有用
>
> 我以前也干过这事
> 觉得AI开得越多，产出越快
> 一个写文案，一个写脚本，一个写朋友圈，齐活儿
>
> 后来发现，我就是在自嗨
> 稿子一堆，能用的没几条。
>
> 为什么？
> 因为每一条都缺一个东西——人的感受
>
> AI能模仿语气、组织逻辑、堆砌信息
> 但它没法替你判断：这句话对不对味？这个角度能不能打动人？
>
> 效率是一个陷阱
> 你觉得自己加速了，其实可能只是把没用的东西生产得更快

### 可复用结构

```text
用工具数量与真实价值的反差作标题
→ 承认自己曾经也采用过这种做法
→ 写当时为什么觉得它有效
→ 呈现产出数量和可用率之间的真实落差
→ 追问无效的根因
→ 区分AI擅长的生成能力与人必须承担的判断
→ 重新定义真正的效率
→ 用一句风险提醒收束
```

### 标题公式

```text
同时使用【多个工具】不等于更高效，真正该看的是【可用结果指标】
```

或：

```text
稿子变多以后，我才发现自己只是把【无效问题】加速了
```

### 这条文案有效的原因

1. 从本人走过的弯路切入，观点不是站在高处指责别人。
2. “稿子一堆、能用的没几条”直接区分产出量和有效产出。
3. 明确提出人仍需负责语感、角度和受众判断，符合人机协作现实。
4. “把没用的东西生产得更快”能够形成对虚假效率的记忆点。

### 风险与使用边界

- “十几个AI”必须是实际数量或明确的概数，不能为了标题冲击力随意夸大。
- “人的感受”不是唯一原因，无效内容还可能来自输入不清、缺少客户资料、目标错误和工作流设计不当。
- AI可以协助分析语气、受众和共鸣风险，但没有使用者的亲身经历、价值责任和最终决定权；不宜绝对说AI完全不能判断。
- 多智能体本身不是问题，在任务边界清楚、有质量门槛和汇总机制时也可能有效。
- 真正效率应同时观察可用率、修改时间、发布结果和业务反馈，而不只看生成速度。
- 不应从个人失败经验推出“开多个AI都没用”的普遍结论。
- 批量生成过程中要避免把客户资料、聊天记录和商业机密无边界地分发给多个工具。

### 与三三当前业务的适配判断

- 高度适配：不用AI制造更多无效碎片，要让AI进入业务闭环。
- 高度适配：AI负责生成、整理和辅助分析，人负责事实、感受、价值判断和最终决策。
- 需要补充：用真实采用率、修改时间和业务反馈定义“有效”。
- 不应得出：工具越少越好，或AI无法辅助内容判断。

### AI 生成时需要提取的变量

- 实际同时使用了多少工具或智能体；
- 每个工具承担什么任务；
- 总共产生多少内容，最终采用多少；
- 被淘汰内容的主要原因；
- 使用者本人必须加入哪些经历、判断和语感；
- AI可以继续辅助哪些分析和检查；
- 真实修改时间、采用率和发布结果；
- 多工具之间是否有路由、汇总和质量门槛；
- 是否涉及客户资料与隐私扩散；
- 使用者对“真正效率”的衡量标准。

## 案例 10｜停止追逐大模型并转向业务落地

### 案例类型

- 圈型：认知圈（产品定位型）
- 子类型：工具焦虑、模型选择与业务变现
- 素材性质：用“追模型”和“用现有模型做业务”的反差，提醒读者把注意力从工具更新转向真实业务应用
- 适用场景：使用者已有使用常见大模型完成业务任务和获得付费结果的真实案例，希望帮助客户减少工具选择焦虑

### 核心主张

- 对许多常见业务任务而言，持续更换大模型不一定是当前最大瓶颈。

- 模型能力与业务结果之间仍需要方法、数据、人工判断和持续迭代。

### 证据状态

- 当前只收到观点型成品文案，没有收到具体业务、成交金额、模型任务、客户案例或对比测试。
- “已经开始收钱”“疯狂收钱”“99%的人”均没有准确口径和证据，不能视为普遍事实。

### 可复用结构

```text

→ 解释哪些任务不需要持续追新模型
→ 说明什么情况下模型差异仍然重要
→ 把注意力带回客户、流程、执行和反馈
```

### 标题公式

```text
别再只比较【工具】，先用手上的模型完成第一个【真实业务任务】
```

```text
模型更新很快，但你当前真正卡住的可能是【业务环节】
```

### 这条文案有效的原因

1. “追模型”和“收钱”形成强结果反差，能抓住工具焦虑人群。
2. 点名常见模型，降低读者对必须拥有复杂工具栈的恐惧。
3. 最终回到业务落地，与三三当前定位一致。
4. 内容短，观点集中，适合作为品牌立场型朋友圈。

### 风险与使用边界

- 提及豆包、GPT等品牌时，应作为工具示例，不暗示官方合作或背书。
- 与案例08主题相近，最终封装时可合并为“从工具焦虑到业务应用”模板组，避免重复。

### 与三三当前业务的适配判断

- 高度适配：先问解决哪个业务环节，再问使用哪个工具。
- 高度适配：用够用的模型完成真实任务，而不是不断囤积工具。
- 需要修改：从“任何模型都够用”改成“多数常见任务先选一个满足要求的模型开始验证”。
- 需要修改：从“疯狂收钱”改成可核验的客户行动、效率或成交案例。
- 不可承诺：只要使用某个大模型就能变现。

### AI 生成时需要提取的变量

- 目标客户正在比较哪些模型；
- 客户真正要完成的业务任务；
- 该任务对模型能力、隐私和工具有哪些要求；
- 使用者实际选择了哪个模型及原因；
- 完成了什么可验证的业务动作；
- 是否产生付费、咨询、作品或效率结果；
- 结果的金额、周期和证据口径；
- 哪些结果来自模型，哪些来自产品和人工执行；
- 什么情况下应该更换或升级模型；
- 如何在不制造优越感的情况下推动读者行动。

## 案例 11｜为什么要把AI接进现有生意

### 案例类型

- 圈型：认知圈（趋势定位型）
- 子类型：AI进入业务、效率升级与能力放大
- 素材性质：面向2026年下半年，解释AI为什么应该进入展示、产品和客户承接等真实业务环节
- 适用场景：使用者希望建立“AI不是炫技工具，而是业务杠杆”的品牌立场，并能提供具体应用场景
- 配图建议：AI进入业务流程的简图、真实应用案例或任务前后对比

### 核心主张

- AI的价值不在于展示技术，而在于帮助业务更快完成内容、产品和客户承接等任务。
- 当原有业务方向基本成立、执行效率成为瓶颈时，AI可能形成新的杠杆。
- AI不能脱离客户、产品和成交系统自动赚钱。
- AI会放大已有能力，也可能放大错误方向和无效流程。

### 证据状态

- 当前只收到观点型成品文案，没有收到行业数据、效率对比或具体业务案例。
- “2026下半年”和“一定要接进生意”属于时间判断与行动主张，不是所有行业都成立的统一事实。

### 原成品文案

> 2026 下半年，为什么你一定要把 AI 接进生意里？
>
> 很多人以为，自己赚不到钱，是行业不行、定位不行。
>
> 错了！你已经很努力了，还是没有突破，说明老方法的效率已经不够了。
>
> AI 不是拿来炫技的，是帮你更快展示自己、更快做出产品、更快接住客户。
>
> 记住下面这句话，AI 不能替你赚钱，但会放大真正会赚钱的人。

### 可复用结构

```text
用明确时间和业务问题作标题
→ 写目标客户对增长停滞的常见解释
→ 提出效率可能是其中一个待检查变量
→ 列举AI可以进入的具体业务环节
→ 说明AI发挥作用需要哪些业务前提
→ 补充AI也会放大错误和无效流程的风险
→ 用一句品牌立场收束
```

### 标题公式

```text
在【当前阶段】，哪些生意应该把AI接进【具体业务环节】？
```

或：

```text
AI不能替你【最终结果】，但可以放大已经成立的【业务能力】
```

### 这条文案有效的原因

1. 时间节点能制造当下感，推动读者重新审视现有工作方式。
2. 展示、产品和客户承接都是明确业务环节，不停留在泛泛的AI趋势。
3. “AI不能替你赚钱”主动限制技术神话，符合三三当前定位。
4. 观点简短，适合作为品牌长期主张的入口内容。

### 风险与使用边界

- 赚不到钱可能来自行业、定位、产品、流量、信任、成交或交付问题，不能未经诊断就断定只是效率不足。
- “错了”会把复杂商业问题强行归结为单一答案，建议改成“也可能不是这些，而是效率已经成为新瓶颈”。
- 并非所有业务都必须在2026下半年立即接入AI，应考虑任务价值、成本、数据风险和团队能力。
- “更快做出产品”不能跳过客户验证、质量控制和真实交付设计。
- “放大真正会赚钱的人”容易把未盈利客户排除在外，更准确的是AI会放大已经清晰的能力、流程和判断。
- AI也会放大错误定位、低质量内容和无效流程，接入前需要先确认业务方向。
- 带有年份的内容会过期，Skill生成时必须根据当前日期决定是否保留时间标题。
- 不能把AI提效自动等同于收入增长，仍需观察咨询、成交、交付和复购数据。

### 与三三当前业务的适配判断

- 高度适配：AI不是炫技，而要进入展示、产品和客户承接。
- 高度适配：AI不能代替赚钱系统，只能放大已有业务能力。
- 需要补充：AI同样会放大错误，因此先诊断业务，再选择工具。
- 需要调整：从“所有人一定要接”改为“识别最值得AI介入的业务环节”。

### AI 生成时需要提取的变量

- 内容发布时的真实时间节点；
- 目标客户所在行业和业务阶段；
- 当前增长停滞的可能原因；
- 是否已经验证产品、客户和成交路径；
- 当前最耗时、重复或易出错的业务环节；
- AI能具体完成什么任务；
- AI接入的成本、隐私和质量风险；
- 使用前后的效率和业务指标；
- 哪些判断仍必须由人完成；
- 什么情况下不适合立刻接入AI。
