# 三三AI商业Skill系统蓝图

## 一句话定义

这是一套面向已有产品和真实成交的创业者，把模糊业务卡点识别为当前最大约束，再调用专项 Skill 推进获客、成交、交付和AI工作流的开放系统。

## 不是什​​么

- 不是28个Skill的中文改名版；
- 不是只会推荐工具的菜单；
- 不是用AI替代商业判断；
- 不是把所有方法塞进一个超级提示词；
- 不是免费内容故意说一半的导流器。

## 总体结构

```text
真实业务输入
→ 三三AI商业总入口
→ 准入判断：产品、成交、目标和证据
→ 根因判断：症状、最大瓶颈、AI适配
→ 一个已上线专项Skill
→ 真实执行
→ 结果数据回流
→ 重新判断下一步
```

## 四层能力

### 1. 入口层

负责新手引导、任务前路由、任务后导航和未覆盖需求登记。V1：`sansan-business-router`。

### 2. 诊断层

负责从多系统症状中找根因、优先级、承接能力和AI适配点。V1：`sansan-business-scan`。

### 3. 执行层

完成一个具体业务任务，并产生可回流的业务结果。V1：`sansan-wechat-moments-coach`。后续按真实需求增加成交、内容、产品、获客、交付和AI工作流专项。

### 4. 状态与质量层

保存事实、结论、执行结果、版本与复盘；检查真实性、权限、版权、路由冲突和安全。V1先使用报告结构与仓库校验，真实重复使用后再做持久化脚本。

## 八个业务观察面

客户资产、产品结构、内容与信任、获客、成交路径、交付与复购、AI增长效率、创始人战略。

八个观察面用于定位，不强迫每个专项 Skill 以八大系统命名。一个 Skill 可以跨多个系统，但必须说明主要业务结果和输入边界。

## V1闭环

```text
“我业务哪里有问题？”
→ 商业总入口
→ 商业系统扫描
→ 最大瓶颈是内容与信任，7天动作需要真实朋友圈素材
→ 朋友圈内容教练
→ 发布并回流私信/咨询/成交数据
→ 重新扫描
```

V1无法处理的需求会形成需求卡，不会被假装路由到不存在的 Skill。

## 当前产品承接假设

根据本次对话，正在设计的路径是：免费开源 Skill 系统 → 199元/年本人群内答疑 → 9999元主动扫描与个性化商业重构。

但项目外的现行商业总纲仍记录3980/19800/39800/10万元四阶段体系。两者尚未完成正式业务真源切换，因此本仓库只把新路径标记为“设计假设”，不在公开README中把9999写成正式在售产品。
