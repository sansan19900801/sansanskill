# 用户问题分类与路由规则

## 路由先后顺序

1. **授权与安全**：是否涉及发布、付款、隐私、外部系统或高风险承诺。
2. **业务准入**：是否有产品，是否有过真实成交。
3. **任务类型**：诊断、决策、创作、执行、记录或技术基建。
4. **业务结果**：主要影响获客、成交、交付、复购还是创始人效率。
5. **问题层级**：表面症状、直接原因、根因或资源约束。
6. **AI适配**：能直接AI化、先补标准、先补素材或必须人工主导。
7. **能力可用性**：只选择注册表中 `ready` 的 Skill。

## V1路由表

| 用户信号 | 当前路由 | 不应误路由 |
| --- | --- | --- |
| 有一个具体商业问题，需判断问题是否成立 | 商业诊断问诊 | 直接顺着用户前提给建议 |
| 多环节混乱、不知先解决什么、会AI无结果 | 商业诊断体检 | 直接推荐工具或写内容 |
| 明确要做业务体检、找最大瓶颈、能力缺口 | 商业诊断体检 | 平均给八个模块建议 |
| 提交观点、案例、实验或外部方法论要求沉淀 | 商业原子收录 | 要求用户自己记字段或直接写入未确认材料 |
| 真实朋友圈素材、客户截图、观点或生活事件 | 朋友圈内容教练 | 商业全盘扫描 |
| 只说“帮我看看”且无背景 | 总入口只问一项真实任务 | 展示完整Skill菜单 |
| 私聊成交、产品定价、短视频等未上线专项 | 输出需求卡，可给一次性通用建议 | 虚构专项Skill |

## 路由冲突处理

- 用户说“帮我写朋友圈”，即使背后商业系统可能有问题，也先尊重明确任务；只有素材明显无法支持目标时说明根因。
- 用户说“我没流量，要写100条朋友圈”，先判断他要的是执行还是诊断；若不知道为何无流量，优先扫描。
- 用户说“用AI帮我成交”，先判断成交路径和输入标准；没有标准时不能直接自动化。
- 用户完成一个专项任务后，先收集结果，不为了继续使用系统而强行推荐下一个Skill。
